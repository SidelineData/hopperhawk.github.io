Website, Blog, and v1.4 Launch!

News November 11, 2024

✎ Edit Post

Welcome! I have been working on HopperHawk on and off in my (limited) free time for a while now, and have finally reached a point where I can share it with others! In this post I will talk about the current hardware and my plans for version 2.0

Version 1.4

Specifications

  • Based on the ESP32
  • Support for an 18650 Lithium Ion battery
  • Uses the TOF10120 sensor for measurements
  • Has a BCM to support charging/load sharing
  • Includes two onboard LEDs to indicate power and charging status
  • USB-C port for power

Limitations

  • The USB-C port is power only, no data!
  • If you need to program/flash the board you’ll have to use a USB TTL cable and a jumper (to put into PROG mode)
  • The LEDs are connected directly to voltage regulator and BCM, no way to adjust/toggle/configure them
  • The LEDs are also poorly located and can be hard to see
  • The board has a misprint on the bottom side of it (which you’ll never see in the case) that indicates the wrong model number/date and a useless QR code

Why is it like this?

HopperHawk started as a project for myself, and I never intended on releasing it out into the world. It’s also the first project that involved building custom PCBs, so when I started out I went as simple as possible to reduce complexity and cost (hence the lack of a serial communication chip). The project got some traction and more people were interested in it than I expected, so I am working on making it better :) Aside from the inconvenience of having to use a special cable to program the board, there is nothing functionally wrong with this batch so I decided to sell them at a discount (basically at cost) while I work on the changes for model 2.0


Version 2.0

What to expect

For the most part, model 2.0 will be a minor upgrade with some modern conveniences added. It should be the same size and form factor making it backward compatible with the existing case that I designed. While still a work in progress, here are some of the main changes I am implementing:

  • Ditch the pins for programming and handling it all through the USB-C port
  • Adding 90 degree surface mount LEDs for better visibility
  • Making the LEDs programmable
  • Adding a built in temperature sensor so you can get alerted if the hopper is getting too hot for the battery (which shouldn’t happen unless you have bigger issues)

When to expect it

I’m working on the hardware designs now, and expect to be placing in the next few weeks! As long as there are no major issues, my hope is to have them available for purchase by mid-December.
